# Modélisation d'une DB relationnelle
Nous avons déjà brièvement vu ce qu'est une base de données, et que celle-ci est constituée de tables qui vont contenir nos données.

Apparaissent ici deux mots qu'il convient de relever :
## 1. Modélisation / Modèle 
#def une **modélisation** : la conception et l'utilisation d'un modèle
#def un **modèle** : [c'est une représentation d'une réalité](https://fr.wikipedia.org/wiki/Mod%C3%A8le).  
- C'est une vue simplifiée de celle-ci, visant à répondre à certaines questions.
- Un modèle est réalisé en utilisant certaines techniques. 

Dans la suite de ce cours, nous modéliserons nos bases de données à l'aide d'une technique de modélisaiton appelée **modélisation Entités / Associations** (ou encore, modélisation E/A)
## 2. Relationnelle 
L'on parle de **base de donnée relationnelle** pour exprimer le fait que les tables qui constituent ces bases de données sont reliées les unes aux autres.
C'est cette capacité de relier des informations réparties entre différentes tables qui confère aux bases de données relationnelles toute leur puissance.
